# 小厨房点菜单微信小程序需求文档

## 1. 项目概述

本项目是一款个人 / 家庭场景下使用的点菜做菜记录 App，以微信小程序为主要载体。用户可以维护自己的菜品库，浏览菜品做法，将菜品加入今日点菜单，根据点菜单进入做菜台按步骤制作，完成后上传成品图、评分并记录心得，最终形成个人历史做菜记录。

v1 版本优先满足个人账号使用，不实现家庭成员协作共享，但数据结构和页面命名应预留后续家庭共享扩展空间。

## 2. 目标用户与核心价值

### 2.1 目标用户

- 经常在家做饭，希望整理个人菜谱的人。
- 家庭成员中负责点菜、备菜、做菜的人。
- 想记录每次做菜效果、评分和改进心得的人。

### 2.2 核心价值

- 把“想吃什么、怎么做、这次做得怎么样”串成闭环。
- 用户维护的是自己的菜品库，而不是公共菜谱广场。
- 每次做菜都可沉淀为历史记录，方便复盘和下次改进。

## 3. 核心用户流程

```text
用户登录
  ↓
进入首页
  ↓
浏览 / 新增菜品
  ↓
查看菜品详情
  ↓
加入点菜单
  ↓
确认今日菜单
  ↓
进入做菜台
  ↓
按步骤做菜
  ↓
完成制作
  ↓
上传成品图
  ↓
评分和写心得
  ↓
形成个人历史记录
```

## 6. 功能需求

### 6.1 登录

登录页需要支持：

- 微信登录。
- 邮箱 + 密码登录。
- 邮箱 + 密码注册。
- 退出登录。
- 登录态持久化。

登录成功后进入首页。不同用户之间的数据必须隔离。

### 6.2 首页

首页用于承接用户每天进入小程序后的主要操作。

需要包含：

- 顶部问候标题，例如“今天想做点什么？”
- 搜索入口，支持按菜名、食材、做法搜索。
- 今日点菜单摘要：
  - 已选菜品数量。
  - 用餐人数。
  - 预计耗时。
  - 查看详情入口。
- 今日推荐菜：
  - 展示菜品图、名称、难度、耗时、评分。
  - 支持查看做法。
  - 支持加入点菜单。
- 最近做过：
  - 展示最近成品记录。
  - 支持进入历史详情。
- 快捷入口：
  - 我的菜品库。
  - 新增菜品。
  - 做菜台。
  - 历史记录。

### 6.3 菜品库

菜品库展示当前登录用户维护的所有菜品。

需要包含：

- 搜索框。
- 分类筛选：
  - 全部。
  - 荤菜。
  - 素菜。
  - 汤类。
  - 主食。
- 条件筛选：
  - 难度。
  - 时间。
  - 口味。
  - 最近评分。
- 新增菜品入口。
- 菜品列表：
  - 菜品图。
  - 菜名。
  - 难度。
  - 预计耗时。
  - 最近评分 / 平均评分。
  - 查看做法。
  - 加入点菜单。
  - 更多操作入口。

### 6.4 新增 / 编辑菜品

新增和编辑菜品共用表单。

字段包括：

- 菜品名称。
- 菜品分类。
- 菜品图片。
- 简短描述。
- 难度。
- 预计耗时。
- 适合人数。
- 口味标签。
- 食材清单：
  - 主料。
  - 辅料。
  - 调料。
- 做法步骤：
  - 步骤标题。
  - 步骤说明。
  - 步骤图片。
  - 火候。
  - 预计时间。
  - 注意事项。
- 做菜小贴士。

### 6.5 菜品详情

菜品详情页展示完整做法。

需要包含：

- 顶部菜品大图。
- 菜名、描述、标签。
- 预计耗时。
- 适合人数。
- 加入点菜单按钮。
- 编辑菜品按钮。
- 食材清单，按主料、辅料、调料分组。
- 做法步骤列表。
- 每个步骤展示：
  - 序号。
  - 标题。
  - 描述。
  - 火候。
  - 时间。
  - 注意事项。
- 做菜小贴士。

### 6.6 点菜单

点菜单展示用户当前已选择但尚未完成的今日菜单。

需要包含：

- 今日日期。
- 用餐人数。
- 已选菜品数量。
- 预计总耗时。
- 已选菜品列表：
  - 菜品图。
  - 菜名。
  - 备注。
  - 份数。
  - 预计耗时。
  - 制作状态。
  - 排序拖拽手柄。
- 调整顺序。
- 继续加菜。
- 提交菜单。

点菜单提交后进入做菜台。未提交时仍可继续调整菜品、份数、备注和顺序。

### 6.7 做菜台

做菜台是今日菜单的执行页。

需要包含：

- 顶部状态切换：
  - 待制作。
  - 制作中。
  - 已完成。
- 状态统计：
  - 待制作数量。
  - 制作中数量。
  - 已完成数量。
- 菜品卡片：
  - 菜品图。
  - 菜名。
  - 当前状态。
  - 预计耗时。
  - 当前步骤进度。
  - 剩余时间。
  - 开始制作 / 继续制作 / 上传成品 / 查看做法按钮。
- 厨房计时器入口。
- 做菜提示栏。

### 6.8 步骤制作页

步骤制作页用于按步骤完成单道菜。

需要包含：

- 返回入口。
- 菜品缩略图、菜名。
- 当前步骤进度，例如“第 2 步 / 共 4 步”。
- 预计剩余时间。
- 横向步骤进度条。
- 当前步骤卡片：
  - 步骤标题。
  - 步骤图片。
  - 操作说明。
  - 火候。
  - 时间。
  - 注意事项。
- 上一步。
- 下一步。
- 标记完成。

当最后一步标记完成后，菜品状态变为已完成，并引导用户上传成品。

### 6.9 上传成品

上传成品页用于保存本次做菜结果。

需要包含：

- 当前菜品信息。
- 成品照片上传，至少 1 张，最多 3 张。
- 实际耗时。
- 口味反馈：
  - 刚好。
  - 偏淡。
  - 偏咸。
  - 偏甜。
- 制作心得，限制 200 字以内。
- 是否加入历史记录开关。
- 提交成品按钮。

提交后进入成品评分页，或根据后续配置直接回到做菜台。

### 6.10 成品评分

成品评分页用于记录本次做菜效果。

评分维度：

- 味道。
- 外观。
- 还原度。
- 火候。
- 满意度。

需要包含：

- 菜品信息。
- 烹饪用时。
- 完成时间。
- 五维星级评分。
- 综合评分，基于五项评分自动计算。
- 我的评价，限制 100 字以内。
- 稍后再评。
- 提交评分。

### 6.11 我的

我的页面展示个人统计和设置入口。

需要包含：

- 用户头像、昵称。
- 我的菜品数量。
- 做菜记录次数。
- 平均评分。
- 快捷入口：
  - 我的收藏。
  - 历史记录。
  - 评分记录。
  - 新增菜品。
- 最近成品。
- 设置列表：
  - 个人资料。
  - 数据管理。
  - 消息提醒。
  - 帮助与反馈。

### 6.12 历史记录

历史记录展示用户已完成的做菜记录。

需要包含：

- 日期筛选。
- 菜名搜索。
- 分类筛选。
- 历史记录列表：
  - 成品图。
  - 菜名。
  - 完成时间。
  - 实际耗时。
  - 综合评分。
  - 心得摘要。
- 历史详情页。

### 6.13 评分记录

评分记录展示用户曾经评分过的成品。

需要包含：

- 按评分排序。
- 按日期筛选。
- 评分列表。
- 五维评分详情。
- 文字评价。

## 7. 前端技术方案

### 7.1 技术栈

前端使用：

- `uni-app`
- `Vue 3`
- `TypeScript`
- `Pinia`
- `SCSS`

目标端优先为微信小程序 `mp-weixin`。

### 7.2 前端组件库策略

由于项目要求高保真还原，前端实现遵循“组件优先，样式补齐”原则：

1. 能用成熟组件库稳定实现的基础控件，优先使用组件。
2. 组件默认样式无法还原原型时，通过主题变量和 SCSS 覆盖。
3. 业务强相关的菜品卡片、今日菜单卡片、步骤卡片、厨房插画氛围等，用自定义组件实现。

优先使用的组件与能力：

| 场景 | 推荐实现 |
| --- | --- |
| 基础 UI | `wot-design-uni` |
| 表单 | `wd-form`、`wd-input`、`wd-textarea`、`wd-picker` |
| 按钮 | `wd-button`，结合 SCSS 做暖橙渐变 |
| 搜索 | `wd-search` |
| Tab | `wd-tabs`、`wd-tabbar` |
| 筛选 | `wd-drop-menu`、`wd-sort-button` |
| 上传 | `wd-upload` |
| 图片 | `wd-img`、`wd-swiper` |
| 评分 | `wd-rate` |
| 弹窗 | `wd-popup`、`wd-message-box`、`wd-action-sheet` |
| 提示 | `wd-toast` |
| 长列表 | `z-paging` |
| 点菜单排序 | `uni-app movable-area/movable-view` |

### 7.3 自定义业务组件

需要封装以下业务组件：

- `AppPage`：页面外层容器，统一安全区、背景色和页面间距。
- `AppNavbar`：顶部导航和标题区域。
- `BottomTabbar`：高保真底部浮层导航。
- `SectionTitle`：带橙色竖线的模块标题。
- `DishCard`：菜品卡片。
- `DishListItem`：菜品库列表项。
- `TodayMenuSummary`：今日点菜单摘要卡。
- `CookStatusCard`：做菜台状态统计卡。
- `CookDishCard`：做菜台菜品状态卡。
- `StepProgress`：做菜步骤横向进度条。
- `StepCookCard`：当前步骤制作卡片。
- `UploadPhotoGrid`：成品图上传区域。
- `RatingRow`：五维评分行。
- `EmptyState`：空状态。
- `BottomSafeBar`：底部固定操作栏。

## 8. 视觉规范

### 8.1 整体风格

整体风格为温暖、轻松、家庭厨房感。视觉应高保真沿用原型：

- 暖橙作为主品牌色。
- 绿色用于成功态和健康提示。
- 白色和浅暖色作为背景。
- 卡片使用柔和边框和阴影。
- 页面插画以厨房、锅具、蔬菜、食物为核心元素。
- 底部导航使用圆角白色浮层。

### 8.2 色彩建议

| 用途 | 颜色 |
| --- | --- |
| 主色 | `#FF6A1A` |
| 主色渐变起点 | `#FF8A2A` |
| 成功色 | `#67A83B` |
| 警告色 | `#FFB020` |
| 文字主色 | `#2A160E` |
| 文字次色 | `#7A6A60` |
| 页面背景 | `#FFFDFC` |
| 卡片背景 | `#FFFFFF` |
| 浅橙背景 | `#FFF4EA` |
| 边框色 | `#F3DCCB` |

### 8.3 尺寸与布局

- 设计宽度按 `750rpx` 适配。
- 页面左右边距建议 `32rpx`。
- 卡片圆角建议 `24rpx` 到 `32rpx`。
- 主按钮高度建议 `88rpx`。
- 底部固定栏必须处理安全区。
- 图片比例保持稳定，避免列表滚动时布局跳动。
- 长标题、按钮文案和备注必须做换行或省略保护，不能溢出容器。

- 用户可以通过微信登录或邮箱密码登录。

- 底部 Tab、卡片、按钮、图片比例和状态标签需接近原型。
- 暖橙主色和厨房插画氛围需保持统一。
- 主要页面在微信开发者工具中无明显错位。
- iPhone 常见尺寸下无文字溢出、按钮遮挡、底部安全区覆盖问题。

### 12.3 交互验收

- 搜索、分类、筛选、分页加载可用。
- 上传图片可预览、删除和限制数量。
- 星级评分可交互，综合评分自动更新。
- 点菜单排序可拖拽或通过明确控件调整。
- 表单提交有加载态、成功态和错误提示。

## 13. 重要约束与默认假设

- 当前项目目录只有原型图，没有现成源码。
- 实现时需要从零创建 `uni-app` 前端工程和 Node 后端工程。
- v1 个人优先，不做家庭共享协作。
- 高保真还原优先级高于组件默认样式。
- 前端优先使用成熟组件库，避免重复造基础控件。
- 图片先使用服务器本地存储，后续可替换为对象存储。
- 邮箱登录采用邮箱 + 密码，不采用邮箱验证码。
- 示例菜品应是用户自己的初始数据，用户可编辑和删除。
